Compilation

To compile my project, you need to first install the GNU MP Bignum Library (GNU GMP). On the CS50 Appliance, that is quite easy. Simply type the following in the Terminal: 

    sudo yum install gmp-devel

You also need to install Check: 

    sudo yum install check-devel

After GNU GMP is installed, simply type make in the Terminal in the project's top-level directory (the one containing the Makefile).

Configuration

Other than installing GNU GMP, which you've already done, no special configuration is needed.

Use

Here is the usage string for my project:

    project [-t] [-n number_code] d

d is the only required argument. This specifies the position of the first digit to the right of the decimal place that you want to compute.

number_code is an optional argument and has two valid values:

  • pi for computing hexadecimal digits of pi.
  • log2 for computing binary digits of log(2).

The default value for number_code is pi.

t is an optional flag. If set, the program reports how long the computation took.

Examples

Let's compute the millionth hexadecimal digit of pi:

    jharvard@appliance (~/Dropbox/project/quadrillionth-decimal-place): ./project -n pi 1000000
    Hex digits of pi beginning at position 1000000: 26C65E52

Since the default value of n is pi, there is a shorter, equivalent invocation: 

    jharvard@appliance (~/Dropbox/project/quadrillionth-decimal-place): ./project 1000000
    Hex digits of pi beginning at position 1000000: 26C65E52

Now, using the t flag, let's see how long it takes our program to compute the millionth digit of pi: 

    jharvard@appliance (~/Dropbox/project/quadrillionth-decimal-place): ./project -t 1000000
    Hex digits of pi beginning at position 1000000: 26C65E52
        Time to compute = 00:00:11.

11 seconds! Pretty darn fast!!

Finally, let's compute the millionth binary digit of log(2): 

    jharvard@appliance (~/Dropbox/project/quadrillionth-decimal-place): ./project -t -n log2 1000000
    Binary digits of log(2) beginning at position 1000000: 11010100
        Time to compute = 00:00:02.
